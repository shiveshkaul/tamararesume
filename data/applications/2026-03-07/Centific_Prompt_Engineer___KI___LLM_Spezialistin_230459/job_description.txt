Applied Research Scientist – Clinical Documentation Prompt Engineer

This role is Temporary until end of June with possible renovation until December 2027 or longer.

Roles and Responsibilities

Design and optimize clinical prompt templates to produce consistent, specialty appropriate clinical documentation from transcripts.
Run prompt experiments and evaluations (e.g., A/B comparisons, targeted worksets), analyze failure modes, and translate findings into prompt improvements and release-ready configurations.
Operationalize prompt updates through Git and Azure DevOps (ADO): manage work items/bugs, participate in PR workflows, and maintain prompt/config artifacts used in releases.
Incorporate feedback from customers and internal customerfacing teams into an iterative rollout plan spanning multiple specialties, with clear monitoring and follow-up.
Build and maintain Linux/Python research scripts to run inlab experiments, batch evaluations, and data processing to support prompt development.

Qualifications

Experience developing and refining LLM prompts (instruction design, templates, fewshot patterns, and systematic iteration), ideally for semi-structured clinical text generation.
Proficiency in Python and Linux for building scripts/pipelines to run experiments and evaluations.
Experience working with version control and engineering workflows (e.g., GitHub) and ADO for work tracking, PRs, and release coordination.
Strong analytical skills: ability to define evaluation criteria, review outputs, and prioritize fixes based on impact and risk.
Experience with clinical documentation workflows and data.
Clear communication and cross-functional collaboration skills.
Language fluency in English and French, German, or Dutch.

In compliance with the General Disability Law, the submission of the corresponding disability certificate will be valued.
